Introduction to OpenBSD

OpenBSD is a security-focused, free and open-source, Unix-like operating system based on the Berkeley Software Distribution (BSD). Theo de Raadt created OpenBSD in 1995 by forking NetBSD 1.0.[4] According to the website, the OpenBSD project emphasizes "portability, standardization, correctness, proactive security and integrated cryptography."

Special features of OpenBSD:

Assume you have a server running a different operating system than OpenBSD that has been hacked through SQL injection. The attacker may cause havoc on the system if that SQL server was operating as a normal user. The user www, a restricted account, operates OpenBSD's built-in web server. It is also operated within a chroot prison. Finally, it is executed using a shell that does not allow logins. The attacker does not even have access to a shell prompt from which to perform commands. Other operating systems allow chroots, although they are seldom, if ever, used by default. [chroot: chroot works by​​ changing the root directory of a program, restricting access and visibility, and thus providing an added layer of isolation and security.]

Write XOR Execute:

The next security feature pioneered by OpenBSD is known as write XOR execute (W X). A process's or the kernel's address space can be readable or executable, but not both. In 2003, OpenBSD was the first operating system to include this capability in version 3.3.

Guard Pages:

Similarly, guard pages were incorporated into OpenBSD in 2003. Guard pages insert an unreadable and unwritable page in memory at the end of each page of memory to detect overruns.

Address Space Randomization:

This feature eliminates the need for code to be in the same location each time a programme executes. An attacker cannot access data by using a known offset.

Let's imagine you have two programmes, A and B. If programme B has a memory leak and attackers know that programme A is loaded before B in memory, they can use the vulnerability in programme B to crash programme A by writing to its memory area.

If programme A starts before programme B by default, this does not imply that B will follow A in memory. In reality, a wide gap between the two programmes may be created, or programme C could be inserted between A and B. Even if a third-party piece of software, such as an Apache web server, contains a fault, crashing it does not allow the attacker to exploit anything.

Pledge and Unveil:

Pledge and Unveil are two sides of the same coin: Pledge is used to limit filesystem access, while Unveil is used to limit system calls. The unique pairing of Pledge and Unveil makes it difficult to meaningfully compromise a programme. Even if a programme is compromised, the hacker can only write to one file or directory, or call certain systems.

Installation Process

Step 1:

Once you've selected a mirror, you will be brought to a page with several directories. To download the most recent disk image, select the highest release number. The "7.3" folder should include the most recently updated version as of 7th April 2023, but if you see anything newer, use that.

Step 2:

Once in the most recent version directory, there should be subdirectories for different system architectures. For most users, amd64 or i386 images will be the most useful.

Step 3:

Among this set, the "install73.iso" file represents the installation image for version 7.3 of OpenBSD. Click on this file to download and save it onto your system.

Step 4:

Prepare a VirtualBox if not installed already by using the command "sudo apt install virtualbox"

Once VirtualBox is downloaded, installed, and opened, click the "New" button at the top of the window to begin creating the new virtual machine.

Step 5:

The next portion of the virtual machine configuration is memory allocation. The user can select the amount of memory required based on the amount of RAM available on the host machine, else the user can choose to go by the mentioned default value and click Next

Step 6:

On the Hard disc stage, select "Create a virtual hard disc now." Because OpenBSD will need to be installed within the virtual system, a virtual hard drive will be required. Select "Create."

Step 7:

If you don't need to transport the virtual machine image across virtualization tools, leave VDI selected for the Hard disc file type selection step. Click "Next"

Step 8:

The following option is to use a dynamically allocated or fixed size virtual hard disc. For most people, "Dynamically allocated" is the easiest to use because it requires the least amount of setting. Click "Continue."

Step 9:

Finally, the maximum size and location of the virtual hard disc can be specified. This might be an external hard disc or a specific location on a local hard drive. Because the size is the maximum size to which the virtual hard drive may develop, it may be wise to restrict this amount if there are significant size constraints based on hard disc capacity. Because the base OpenBSD installation would be modest, this restriction might simply be adjusted to a few GB. Select an option and click "Create."

Step 10:

The OpenBSD VM should now be accessible to start from the main VirtualBox menu. Simply click the "Start" button at the top of the interface when the OpenBSD virtual machine is selected to start it.

Once the virtual machine is started, it will request a boot medium to start the virtual machine from!

Step 11:

Click on the folder icon to open a file selector. Click "Add," then browse to and select the OpenBSD ISO image downloaded earlier. Click "Open," then "Choose."

Step 12:

Now, click "Start" to launch the virtual machine. The OpenBSD installer should begin to boot.

Step 13:

On first boot, the OpenBSD image will load a text-based installer environment, as shown in the image below. To begin the installation process, type i and press Enter.

Step 14:

This procedure contains a number of questions and setting choices, each of which is discussed briefly. The first question asks for your desired keyboard layout. To list all possible settings, type L or? and hit Enter, or simply type us or uk to alter the keyboard layout to US or UK English.

Step 15:

Then, give the hostname a name, something like "openbsd" or any name of your choice

Step 16:

The following questions concern network setup. The network settings in a virtual computer are pretty straightforward to configure. As the default network setup settings should be sufficient for most users, each option, beginning with Available network interfaces and ending with DNS domain name, can be responded to by just clicking Enter. Depending on your network arrangement, it may be worth taking greater caution when selecting these parameters during a hardware installation.

Step 17:

The following option, Password for root account, allows the root password to be specified. The "will not echo" string indicates that the password will not be displayed when entered, nor will it be returned to the user. In general, the root password should be strong, especially for a server or internet-connected machine.

Step 18:

The installer asks if you want to start the SSH daemon, or background service, by default after you input the root password twice. While not required, it may be beneficial. Enter to activate it.

Step 19:

The following two questions regard the graphical configuration of the system. If you'd like to use a graphical interface, enable the X Window System by pressing Enter. The next question asks if you would like the X Window System to be started by the xenodm login manager. While this isn't necessary, it will make installing a different desktop environment easier, as shown later in this tutorial. To enable it, type yes and press Enter.

Step 20:

Then, for a standard-level user, provide a name and password. Because it is advisable not to operate as the root user for security concerns, this user will be utilised for most common desktop functions. For most users, blocking root SSH login is also a good idea for security reasons.

Step 21:

For the last question in this set, enter ? to see the available time zones, then type your preferred choice from the list or press Enter to continue with the default selection. Then, so the same thing with the sub-time zone.

Step 22:

Then, the installer will partition the disks. In general, the only disk available for the virtual machine will be fine for the installation — click Enter.

While custom partitioning is possible within the virtual machine environment, the default Whole disc MBR configuration, abbreviated to w, is an excellent choice. To proceed with the installation, press Enter once more.

Step 23:

The following section of the installation allows you to select, download, and install "sets" of packages. Enter http when prompted for the location of the sets to download them from the internet.

Step 24:

You can provide your preferred HTTP proxy and server, although most users will be OK with the default configuration of no proxy.

Step 25:

Sets can then be chosen, or you can simply install them all. To begin installing the sets, press Enter. After the sets have finished installing, press Enter again to choose "done" as the location of the sets.

Step 26:

If a time option appears, just click Enter. Then, the OpenBSD installation should be complete! To start it, click Enter to select "Reboot" and wait for the system to load.

Delete the install file

Step 27:

The system may boot back into the installation screen, most likely because the initial installation DVD is still loaded, rather than the virtual hard drive. In this situation, shut down the OpenBSD VM.

Step 28:

Then, within VirtualBox, select the OpenBSD virtual machine and click "Settings."

Step 29:

Right-click on the ISO installation file listed under Storage Devices in the "Storage" menu and select "Remove Attachment." On the popup, select "Remove" and then "OK" to exit the settings. The virtual computer should then be restarted.

Step 30:

If a login screen such as the one below appears, OpenBSD has installed and booted from its virtual hard drive. Now, we can log in to the OpenBSD desktop environment.
